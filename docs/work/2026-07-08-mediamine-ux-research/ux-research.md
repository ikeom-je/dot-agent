## 発見(ファクト)

### 切り抜き・クリップ作成の作業時間(H1関連)

- AI切り抜きツール(Vidyo.ai)を使っても生成クリップには手動調整が必要で、
  1本あたり10〜15分。10本で2時間以上。
  出典: https://note.com/eitayoutuber/n/ne31f188395b0
- AI自動生成(Klap)は30分素材から10本生成に約10分だが、使用可能なのは4本程度(約40%)。
  出典: https://note.com/eitayoutuber/n/ne31f188395b0
- ポッドキャスト切り抜きのフロー構築は初回約3時間、確立後は約30分/回。
  出典: https://note.com/aiarukurashi/n/nfe40c1efbbc0
- 「探す時間」単体を計測した調査・統計は発見できなかった。

### 競合の機能範囲(H5関連)

- **Descript は Project Browser でトランスクリプトを横断検索できる**(検索対象:
  transcripts、composition名、話者ラベル、プロジェクト名等)。
  出典: https://help.descript.com/hc/en-us/articles/10165829215885-Project-file-search
- Descript は検索結果への一括編集アクション(Copy Search Results 等)も提供。
  出典: https://www.descript.com/blog/article/descript-38-new-features-copy-search-results
- Descript はテキストベース編集で切り出し・書き出しまで完結する編集ツールである。
  出典: https://corp.omake.co.jp/descript完全ガイド!テキスト編集で動画制作が革命的に進化/
- Premiere Pro について、複数シーケンスのトランスクリプト横断検索は公式コミュニティで
  ユーザーから要望として挙がっている(標準機能としては確認できなかった)。
  出典: https://community.adobe.com/t5/premiere-pro-discussions/is-there-a-way-to-search-multiple-transcripts-in-different-sequences-at-once/td-p/13224858

### 価格帯(H3関連)

- Notta: 年払いで月1,185円〜(AI要約・話者識別・月1,800分込み)。
  出典: https://www.generativeai.tokyo/media/gijilog2025/
- Rimo Voice: 個人向け文字起こしプラン月1,650円(年払い月1,100円)、2025-05-29開始。
  出典: https://guide.rimo.app/ja/articles/11402332 / https://rimo.app/about/voice/plans

### 文字起こしのコスト・話者分離(H6関連)

- OpenAI Whisper API: 約$0.006/分 ≒ $0.36/時(約50〜60円/時)。Groq 経由では
  約$0.04/時の低価格経路も存在。
  出典: https://aipicks.jp/mag/whisper-guide-2026
- 日本語CER(文字誤り率)は一般会話で4〜5%程度。固有名詞・専門用語は弱い。
  出典: https://fullfront.co.jp/blog/generative-ai/sound-generation/whisper-transcription-accuracy-usage-guide/
- Whisper 単体は話者分離非対応。pyannote.audio との組み合わせ(WhisperX 等)が必要。
  出典: https://weel.co.jp/media/tech/whisper/

### 素材を外部に預けることへの抵抗(H4関連)

- 企業向けには「発売前情報を扱う部門にとって漏えい対策が重要」等の一般論は多数あるが、
  **個人クリエイターの未公開収録素材に特化した抵抗感の調査は発見できなかった**。
  出典(一般論): https://sp-jp.fujifilm.com/image_works/tips/column44.html

## インサイト(解釈)

| 仮説 | 判定 | 根拠となる発見 |
|------|------|----------------|
| H1: 場面探しに配信1回30分以上 | **不明**(間接支持のみ) | クリップ作成全体の時間コストは裏付いた(1本10〜15分×本数)が、「探す時間」単体の統計は無い。インタビューで検証 |
| H2: 探せないから再活用を諦める | **不明** | 死蔵の自覚に関する調査は発見できず。インタビューで検証 |
| H3: 月額1,480円の受容 | **不明**(価格レンジ整合は確認) | 競合(Notta 1,185円/Rimo 1,100〜1,650円)のレンジ内であることは確認。ただし支払意思そのものは未検証 |
| H4: 素材を預ける抵抗は障壁でない | **不明** | 個人クリエイター対象の調査が存在しない。インタビューで検証 |
| H5: 横断検索→切り出しの一続き体験は既存に無い | **不明**(棄却寄り) | Descript が横断検索・テキストベース編集・書き出しを同一製品で提供することは確認。ただし「検索結果→前後再生→切り出し」の一続きの体験として成立しているかは文献では未確認。実機トライアルで確定する |
| H6: 個人価格帯でコスト・品質成立 | **不明**(条件付き示唆) | OpenAI Whisper 経路では 50時間 ≒ $18(約2,700円)で価格仮説1,480円を上回る。Groq 経路(約$0.04/時)なら 50時間 ≒ 約300円で成立余地。話者分離・保存・検索のコストと品質は未実測(prfaq の実測計画は維持) |
| H7: 検索0件時の伝わり方 | **不明** | デスクリサーチでは検証不能。プロトタイプ検証へ |

### H5 が棄却寄りであることを受けた示唆(次boltへの論点)

Descript との違いになり得る候補(いずれも新仮説として要検証):
1. Descript は「編集ツール」であり、編集しないファイル(会議録画・配信アーカイブ)を
   全部入れて検索する「アーカイブ検索」用途には重い・高い可能性
2. 日本語の検索・話者分離品質
3. 「預けるだけ」の非編集者向け体験(user-story P1 の編集最小ワークフロー)

   置いた/置かなかった、その時の判断理由は何だったか(H4。仮想条件を出さず実経験を聞く)

   課金を始めた/やめたツールとその理由は何か(H3。金額を提示せず支出実態から測る)
6. (Descript を知っている場合)なぜ使っている/使っていないのか(H5 再定義の材料)
