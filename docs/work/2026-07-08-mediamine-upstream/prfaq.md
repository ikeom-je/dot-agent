# PRFAQ: MediaMine(仮称)— 収録メディアの発言検索・切り出し

## プレスリリース

### 見出し

「あの発言、どこだっけ」が3秒で見つかる — 収録した動画・音声を発言単位で検索して切り出せる「MediaMine」提供開始

### サブ見出し

ポッドキャストや動画を制作する個人・小規模チームのための収録素材検索サービス。素材を放り込むだけで全編が文字起こしされ、言葉で検索して、その場面をそのまま切り出せる。

### 課題

毎週配信するポッドキャスターや動画クリエイターの手元には、数十〜数百時間の収録素材が眠っている(蓄積量・作業時間の数値はすべて想定シーンであり、H1〜H2で検証する)。SNS用のハイライトを作る、過去回の発言を引用する、対談の名場面をまとめる — そのたびに、どのファイルかを思い出し、シークバーを行き来しながら耳で探す。1時間の収録から1分のクリップを作るのに、探す時間だけで30分かかる、というのが本企画の想定する典型シーンである。結果、素材はあるのに「探すのが面倒だから」と過去素材の再活用を諦め、毎回新しく撮ることになる。

### 解決

MediaMine は、動画・音声ファイルをアップロードするだけで全編を話者付きで文字起こしし、全素材を横断して言葉で検索できるようにする。検索結果は該当発言の前後から即再生でき、「ここからここまで」を選んでクリップとして書き出せる。探す時間が「聞き直す30分」から「検索する数秒」になり、撮りためた素材が使える資産に変わる(見出しの「3秒」を含む検索応答時間は体験目標値であり、実現性は H6 の技術検証に含む)。

### 顧客の声(想定)

> 「対談ポッドキャストを3年続けていて、過去回は150本以上。ゲストの発言を紹介したくても探せませんでした。MediaMine に全部入れてからは、キーワードで一発です。先週は過去回の名言だけで総集編を1本作れました。」— 週次配信のポッドキャスター(想定)

### 始め方

1. 収録ファイル(動画・音声)をアップロードする(フォルダごとでも可)
2. 文字起こし完了の通知を待つ(裏で自動処理)
3. 検索して、再生して、切り出す

## FAQ

### 外部FAQ(顧客からの質問)

**Q: 価格は?**
**A:** 月5時間の文字起こしまで無料、月額1,480円で月50時間+全素材の横断検索(価格は仮説 H3)。

**Q: YouTube の自動字幕や既存の文字起こしツール(Notta 等)と何が違う?**
**A:** 既存ツールは「1ファイルの文字起こし」が主目的で、成果物はテキスト。MediaMine は「全素材を横断した検索」と「そのまま切り出し」までを一続きの体験にする(この差別化は仮説 H5。競合調査で要検証)。

**Q: 収録データはどう扱われる? 学習に使われない?**
**A:** 素材と文字起こしは暗号化保存し、AIモデルの学習には使わない。削除すれば復元されない。— これは提供時に守る設計方針(コミットメント)であり、実現方式と例外の有無は要件フェーズで確定する。

### 内部FAQ(社内からの質問)

**Q: なぜ今やるのか?**
**A:** 音声認識の精度向上とコスト低下で「全素材を全部文字起こししておく」ことが個人価格帯で成立し始めた、というのが本企画の前提仮説(H6 に含む)。制作人口の増加トレンドを含め、定量的裏付けは未調査。

**Q: 最大のリスクは?**
**A:** 文字起こしコスト(計算資源)が価格に対して重く、ユニットエコノミクスが成立しない可能性。および YouTube・編集ソフト(Premiere 等)が同機能を内蔵してくること。

**Q: 技術的に一番難しい点は?**
**A:** (1) 話者分離の精度(対談素材で誰の発言かを誤ると検索体験が崩れる)。(2) 長時間素材の処理コストと待ち時間の設計。どちらも答えを持っていない段階であることを明記する。

**Q: 成功指標は?**
**A:** (1) アップロード後7日以内に検索→切り出しまで到達したユーザーの割合 40% 以上、(2) 有料転換率 10% 以上、(3) 週1回以上検索するユーザーの3ヶ月継続率 70% 以上(いずれも仮の目標値)。

**Q: 答えられていない質問(検証すべき仮説)**
**A:** 以下は未検証であり、UX調査(今回は省略、次bolt以降)の仮説とする:
- H1: 対象ユーザー(定期配信する制作者)は過去素材から場面を探す作業に配信1回あたり30分以上かけている
- H2: 彼らは「探せないから再活用を諦める」経験をしている(素材の死蔵が課題として自覚されている)
- H3: 月額1,480円は「探す時間がなくなる」対価として受け入れられる
- H4: 収録素材(未公開含む)を外部サービスに預けることへの抵抗は導入の障壁にならない
- H5: 既存の文字起こしツール・編集ソフトは「全素材横断の発言検索→切り出し」を一続きに提供していない(競合調査で確認)
- H6: 話者分離を含む文字起こしの品質・コスト・検索応答は個人向け価格帯で成立する
  (技術調査。まず対談素材10本で話者分離の誤り率と1時間あたり処理コストを実測して判定する)
