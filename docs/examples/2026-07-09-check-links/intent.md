# intent: check-links

- 解きたい課題: ドキュメントの相対リンク検証を毎回インラインのワンライナーで書いており、
  再現性がなく、bolt の収束チェック(グリーン判定)の正式コマンドが存在しない。
- この bolt でやること: `tools/check-links.sh` を実装する。全 .md の相対リンクの実在を検証し、
  壊れリンクを報告して非0で終了する。テスト付き(TDD)。
  ※ 内部改善boltのため、lifecycle の省略ルールによりフェーズ2〜5(PRFAQ〜UX設計)は省略。
- やらないこと: 外部URLの死活チェック、アンカー(#見出し)の実在検証、CI設定。
- 完了の定義: テストがグリーン(自分で実行して確認)、AGENTS.md のプロジェクト固有欄に
  テストコマンドとして登録、相互レビュー収束、verification.md 記録済みで merge。
