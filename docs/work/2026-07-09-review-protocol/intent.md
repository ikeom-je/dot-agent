# intent: review-protocol

- 解きたい課題: 5回のboltで実証されたクロスレビューの実行手順(レビュー依頼プロンプトの型・
  2往復プロトコル・指摘の再現検証・採否記録)と、委譲失敗時のフォールバック
  (Antigravity空応答→WebSearch等)が、オーケストレーターの暗黙知のままで
  ドキュメント化されていない。別プロジェクト・別セッションで再現できない。
- この bolt でやること:
  1. `cross-review` スキルを新設(レビュー依頼プロンプトのテンプレート、2往復プロトコル、
     指摘の再現検証と採否記録の型、レビュアー不能時のフォールバック)
  2. cli-routing に委譲失敗時のフォールバック規則を追記(リトライ上限は
     新パラメータ `max_delegation_retries` として AGENTS.md に追加)
  3. AGENTS.md 参照表・review.md からのリンク接続
  ※ 内部改善boltのため上流フェーズ(2〜5)は省略。
- やらないこと: review.md の観点・収束条件の変更(実行手順の定式化のみ)、
  レビューの自動化スクリプト。
- 完了の定義: 新スキルが実証済み手順を過不足なく含み、`./tools/check-links.sh` と
  `./tools/test-check-links.sh` がグリーン、相互レビュー収束、merge 済み。
