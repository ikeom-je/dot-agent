# review.md — 相互レビュー(クロスモデルレビュー)の運用

具体的なレビュー依頼の手順・プロンプトの型は
[cross-review スキル](../../.claude/skills/cross-review/SKILL.md) を使う(本docは運用ルール)。

作った本人(モデル)は自分の間違いに気づきにくい。dot-agent では**全フェーズの成果物**
(intent / PRFAQ / UX調査 / ユーザーストーリー / UX設計 / spec / plan / コード)を、
**作ったモデルと別のモデル**が
レビューすることを基本とする。対象フェーズは `review_gate_phases`、往復回数は
`max_review_rounds`([AGENTS.md](../../AGENTS.md) の収束パラメータ)に従う。

## 成果物ミニループ(全フェーズ共通)

1. **計画** — この成果物で何を満たすかを数行で確認する(テンプレートの完了条件・
   スキルのセルフチェックがこれにあたる)
2. **作成** — テンプレート/スキルに沿って作る
3. **検証** — セルフチェック(コードならテスト実行。ループ上限は [test.md](test.md))
4. **相互レビュー** — 下の表のレビュアーに依頼する(`review_gate_phases` の対象フェーズのみ)
5. **収束** — 指摘の採否を決めて反映。往復が `max_review_rounds` に達したら打ち切り、
   未解決指摘を verification.md(上流だけの bolt では intent.md 末尾の引き継ぎメモ)に記録して
   `escalation_to` へエスカレーション

## レビュアーの選び方

原則は2つだけ: **(1) 作った主体と別のエージェントがレビューする。(2) 奏者の成果物は
最終的に必ず指揮者(Claude)が検証する。** 下表は既定編成での対応であり、
[AGENTS.md のエージェント編成表](../../AGENTS.md) を変更したら、この2原則を保ったまま
読み替える(表の名前に縛られない)。

| 成果物を作った主体 | レビュアー |
|---|---|
| Claude(オーケストレーター自身) | Codex または Antigravity(cross-reviewer subagent / antigravity:review) |
| Codex(codex-worker 経由) | Claude 自身がレビュー。大きい diff なら Antigravity で一次レビュー後に Claude |
| Antigravity | Claude 自身がレビュー(必須。Antigravity の出力は必ず Claude が検証する) |

ドキュメント成果物(PRFAQ / spec / plan)も同じ表に従う。cross-reviewer subagent は
コードだけでなくドキュメントのレビューにも使う。

## レビュー観点

**共通(全成果物):**

1. **intent との整合** — 成果物が `intent.md` / 前フェーズの成果物と矛盾していないか、
   範囲を逸脱していないか。範囲外は [git.md](git.md) の分割基準へ。
2. **完了条件の充足** — そのフェーズの完了条件([lifecycle.md](lifecycle.md))を実際に
   満たしているか。
3. **簡潔さ / YAGNI** — 不要な抽象化・重複・膨らみ。

**コード固有:** バグ・エッジケース・エラー処理の欠落、テストを弱める変更の混入([test.md](test.md))。

**ドキュメント固有:** 曖昧で2通りに解釈できる記述、検証不能な主張(出典・根拠なし)、
答えたふりをして答えていない FAQ。

## 収束条件(停止ルール)

- **往復は `max_review_rounds` まで**(レビュー→修正を1往復と数える)。上限で収束しない指摘は、
  verification.md(上流だけの bolt では intent.md 末尾)に「未解決の指摘」として記録し、
  `escalation_to` の判断を仰ぐ。
- **指摘の採否は Claude が最終判断する**。盲目的に全反映しない:
  - 指摘が正しいか、まず検証する(コードを読む・動かす・出典を確認する)。
  - 正しくない/好みの問題と判断した指摘は、理由を添えて棄却してよい。棄却理由は verification.md に残す。
- レビューで新機能・新調査の提案が出ても、この bolt では対応しない。新しい bolt として起票する。
