# spec: check-links

## スコープ / 非スコープ

- スコープ: リポジトリ内の全 `.md` ファイル中の Markdown インラインリンク記法の
  相対パス実在検証。CLI スクリプト+テスト。
- 既知の制限: コードスパン・コードブロック内のリンク記法も検証対象になる
  (ドキュメント側で実在しない例をリンク記法で書かない運用とする)。
- 非スコープ: 外部URL(http/https)の死活、アンカーの実在、参照スタイルリンク等の他記法、CI。
- 画像記法のリンク先は通常リンクと同様に検証する(記法上は区別しない)。
- 空リンク(リンク先が空)は壊れリンクとして報告する。

## 要件

1. `tools/check-links.sh [dir]` で実行できる(引数省略時はカレントディレクトリ)。
2. 対象: `dir` 以下の全 `.md`(`.git/` は除外)。
3. リンクのうち `http://`・`https://`・`#` 始まりはスキップする。
4. 相対リンクは「そのファイルのあるディレクトリ基準」で解決し、`#` 以降は無視して
   実在(ファイルまたはディレクトリ)を確認する。
5. 壊れリンクは `BROKEN: <file> -> <link>` 形式で1行ずつ標準出力に出す。
6. 終了コード: 全リンク有効なら 0(`ALL LINKS OK` を出力)、壊れリンクありなら 1。
7. 依存は bash + 標準コマンド(grep/sed)のみ。

## 成功基準

- テストスクリプト `tools/test-check-links.sh` が、正常系(有効リンクのみ)と
  異常系(壊れリンク・httpスキップ・アンカースキップ)をフィクスチャで検証し、グリーン。
- リポジトリ本体に対して実行して 0 で終了する。
