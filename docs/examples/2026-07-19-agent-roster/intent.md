# intent: agent-roster

プロファイル: standard

- 解きたい課題: エージェントの担当領域(Claude=判断/Codex=実装/Agy=調査)が
  AGENTS.md・cli-routing・review.md の複数箇所にハードコードされており、モデルの強さ・
  得意領域が変わったとき(例: マルチモーダル生成・スライド生成は Gemini が強い)に
  組み替えられない。
- この bolt でやること:
  1. AGENTS.md に「エージェント編成」表を新設(★単一の定義箇所): 能力領域→担当→
     代替(フォールバック順)のマトリクス。マルチモーダル領域(画像生成・検出・比較、
     スライド生成)を新領域として追加し既定担当を Antigravity に
  2. 指揮者(判断・検証・グリーン判定・レビュー最終採否)は Claude 固定と明記
     (権限構造の前提のため。可変なのは奏者の領域)
  3. cli-routing の判断表・フォールバック表を編成表参照に置き換え(重複除去)、
     「編成の変更手順」を追記
  4. review.md のレビュアー表を編成変更に追従する形に注記
  ※ 内部改善boltのため上流フェーズ(2〜5)は省略。
- やらないこと: 指揮者役の可変化、subagent 定義の自動生成、Google Slides API 等の
  具体的なツール接続(領域と担当の定義まで)。
- 完了の定義: 担当領域の定義が AGENTS.md 編成表のみにあり他docは参照(grep で確認)、
  正式グリーン判定グリーン、相互レビュー収束、merge 済み。
