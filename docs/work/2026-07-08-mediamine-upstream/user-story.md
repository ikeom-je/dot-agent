## ペルソナ

- 状況: 副業で対談ポッドキャストを3年・150本配信(未調査)。編集は自分で、週の可処分時間は限られる

### P2: ケンジ・小規模チームの動画ディレクター(仮説)

- 状況: 3人チームでクライアントのインタビュー動画・ウェビナーを制作(未調査)
- ゴール: 納品後の素材から追加クリップの依頼に素早く応えたい
- 現状のやり方: 撮影日とファイル名の規約で当たりを付け、プレビューで探す
- 不満・障害: 「あの人があれを言ったカット」の特定に時間がかかり、追加依頼の利益率が下がる

MVPは P1 に絞る(P2 はチーム共有・権限が必要になり、スライスが厚くなるため「後」に回す)。

## ストーリー詳細(MVPスライスのみ)

### US-1: 素材を預けて文字起こしを待つ

- ストーリー: P1 として、過去素材を検索可能にするために、収録ファイルをまとめてアップロードしたい
- 受け入れ条件:
  - [ ] 動画・音声の主要形式(mp4/mov/mp3/wav/m4a)をアップロードできる
  - [ ] 処理中でも画面を閉じてよく、完了が通知される
  - [ ] 各素材の処理状態(待機/処理中/完了/失敗)が一覧で分かる
- 根拠となる調査結果: なし(H1・H4 の仮説に依拠)

### US-2: 言葉で横断検索する

- ストーリー: P1 として、目当ての発言を見つけるために、全素材を横断してキーワード検索したい
- 受け入れ条件:
  - [ ] 検索結果に「素材名・発言タイムスタンプ・前後の文脈テキスト」が表示される
  - [ ] 結果は複数素材をまたいで返る
  - [ ] 検索実行後、画面遷移や追加操作なしに同一画面で結果が表示される
        (応答時間の数値目標は spec で定義)
- 根拠となる調査結果: なし(H1 の仮説に依拠)

### US-3: 該当箇所を前後から再生して確かめる

- ストーリー: P1 として、それが目当ての場面か確かめるために、検索結果から少し手前を再生したい
- 受け入れ条件:
  - [ ] 検索結果クリックで、該当発言の開始時刻より手前から再生が始まる
        (手前オフセットの値は spec で定義)
  - [ ] 再生位置に同期して文字起こしがハイライトされる
- 根拠となる調査結果: なし(H1 の「探す→確かめる」の想定シーンと、prfaq「解決」の
  前後から即再生する価値に依拠)

### US-4: 範囲を選んでクリップを書き出す

- ストーリー: P1 として、SNS投稿や引用に使うために、確かめた場面を範囲指定して切り出したい
- 受け入れ条件:
  - [ ] 開始・終了点を指定してクリップを作成できる(元素材は変更されない)
  - [ ] 書き出したクリップをダウンロードできる
- 根拠となる調査結果: なし(H2 の仮説に依拠)

## 優先度の判断理由

- MVPスライスは「預ける→探す→確かめる→切り出す」の背骨を一本通す最小構成。
  どれか1つ欠けると PRFAQ の価値(探す時間の削減→再活用)が成立しない。
- 話者絞り込み・意味検索は検索体験を高めるが、キーワード検索だけで価値仮説(H1/H2)は
  検証できるため「次」以降に回した(YAGNI)。
- P2(チーム)は共有・権限・請求の複雑さを持ち込むため「後」。
