# review.md — クロスモデルレビューの運用

実装した本人(モデル)は自分の間違いに気づきにくい。dot-agent では**書いたモデルと別のモデル**が
レビューすることを基本とする。

## レビュアーの選び方

| 実装した主体 | レビュアー |
|---|---|
| Claude(オーケストレーター自身) | Codex または Antigravity(cross-reviewer subagent / antigravity:review) |
| Codex(codex-worker 経由) | Claude 自身がレビュー。大きい diff なら Antigravity で一次レビュー後に Claude |
| Antigravity | Claude 自身がレビュー(必須。Antigravity の出力は必ず Claude が検証する) |

## レビュー観点

1. **正しさ** — バグ・エッジケース・エラー処理の欠落。
2. **intent との整合** — diff が `intent.md` / `spec.md` の範囲に収まっているか。
   範囲外の変更は [git.md](git.md) の分割基準へ。
3. **簡潔さ** — 不要な抽象化・重複・YAGNI違反。

## 収束条件(停止ルール)

- **レビュー往復は最大2回**(レビュー→修正→再レビュー→修正、で打ち切り)。
  2往復で収束しない指摘は、verification.md に「未解決の指摘」として記録し、人間の判断を仰ぐ。
- **指摘の採否は Claude が最終判断する**。レビュアーの指摘を盲目的に全部反映しない:
  - 指摘が技術的に正しいか、まず検証する(コードを読む・動かす)。
  - 正しくない/好みの問題と判断した指摘は、理由を添えて棄却してよい。棄却理由は verification.md に残す。
- レビューで新機能の提案が出ても、この bolt では対応しない。新しい bolt として起票する。
