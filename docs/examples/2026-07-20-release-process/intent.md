# intent: release-process

プロファイル: standard

- 解きたい課題: プロセスが squash merge で終わっており、リリース(判定・タグ・
  リリースノート・ロールバック)の規定が無い(2026-07-20 監査のギャップ1)。また
  lean-canvas テンプレートがどのスキルからも参照されず孤立している(ギャップ3)。
- この bolt でやること:
  1. `docs/process/release.md` を新設: リリース判定チェックリスト、バージョニング/タグ
     規約、リリースノートの作り方(コミットログの Why から集約)、ロールバック方針、
     GO 判断は `escalation_to`(人間)。プロジェクト種別(ライブラリ/サービス/doc)に
     依存しない汎用形とし、固有手順は AGENTS.md プロジェクト固有欄に委ねる
  2. lifecycle.md に「リリース(bolt 横断の節目)」を接続(bolt 内フェーズではなく
     複数 bolt を束ねる節目として)。AGENTS.md 参照表・git.md からリンク
  3. prfaq スキルに lean-canvas への導線を追加(課題抽出・事業構造の俯瞰の任意ステップ)
  ※ 内部改善boltのため上流フェーズ(2〜5)は省略。
- やらないこと: CI/CD 設定・デプロイスクリプトの実装、特定ホスティング/ストアの手順、
  リリースの自動化。
- 完了の定義: release.md が存在し lifecycle/AGENTS.md/git.md から参照され、
  lean-canvas に参照元がある。正式グリーン判定グリーン、相互レビュー収束、merge 済み。
