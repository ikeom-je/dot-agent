# intent: roster-review

プロファイル: standard

- 解きたい課題: エージェント編成表は編集可能になったが、「どのモデルがどの領域に強いか」の
  判断材料を作る手順がない。人間が根拠なしに担当を決めるか、指揮者(Claude)の独断に
  なってしまう。
- この bolt でやること:
  1. `roster-review` スキルを新設: 3モデル(Claude/Codex/Antigravity)に能力領域ごとの
     自己評価をさせ、相互の評価を見せて合意形成(異議は1回)し、編成表の改訂案
     (現行→提案・根拠・合意状況付き)を人間に提示する手順。適用は人間の承認後のみ
  2. 自己申告バイアス対策: 自己申告に加えて、このリポジトリの実績(verification.md の
     記録)と公開情報を判断材料に含める
  3. cli-routing の編成変更手順・AGENTS.md 参照表への接続
  4. スキルを1回実行し、実際の改訂案(roster-proposal.md)を作って人間に提示する
  ※ 内部改善boltのため上流フェーズ(2〜5)は省略。
- やらないこと: 人間の承認なしの編成表書き換え、ベンチマークの自動実行、
  モデルAPIバージョンごとの能力DB化。
- 完了の定義: スキルが存在し参照が接続され、実行1回分の改訂案が人間に提示されている。
  正式グリーン判定グリーン、相互レビュー収束、merge 済み。
