# intent: mediamine-upstream

- 解きたい課題: ポッドキャストや動画を作る個人・小規模チームは、撮りためた収録素材
  (配信アーカイブ・インタビュー音声・会議録画)の中から「あの発言どこだっけ」を
  探すのに、動画をシークしながら耳で聞き直しており、1本の切り出しに収録時間と
  同じくらいの時間がかかっている。
- この bolt でやること: メディア検索・切り出しサービス(仮称: MediaMine)の上流一式を
  作る — PRFAQ → ユーザーストーリー → UX設計。フレームワークの新フェーズ
  (user-story / ux-design)のテスト実行を兼ねる。
- やらないこと: UX調査(Web調査を伴うため今回は省略。lifecycle の省略ルールを適用し、
  ペルソナ・数値は全て仮説と明記する)、要件(spec)以降、技術選定。
- 完了の定義: prfaq.md / user-story.md / ux-design.md の3成果物がそれぞれ
  セルフチェックと相互レビュー(`max_review_rounds` 以内)を収束し、
  引き継ぎメモ付きで merge されている。

## 引き継ぎメモ(上流bolt収束時の記録)

- 相互レビュー: いずれも Codex(read-only)、`max_review_rounds`=2 以内で収束。
  - prfaq.md: 指摘5件(major3・minor2)→ 4件採用+1件部分採用 → 2往復目「収束」
  - user-story.md + ux-design.md(合同レビュー): 指摘5件(major3・minor2)
    → 4件採用+1件部分採用(話者ラベルは表示のみと明記して残置。棄却理由記録済み)
    → 2往復目「収束」
- UX調査は省略(lifecycle の省略ルール適用)。ペルソナ・数値は全て仮説と明記済み。
- 次の bolt への引き継ぎ:
  1. UX調査 bolt: H1〜H6 の検証(特に H5 競合=Notta/編集ソフト、H6 話者分離の実測)。

  2. 要件(spec)bolt: US-2 応答時間・US-3 手前オフセットの数値目標、
