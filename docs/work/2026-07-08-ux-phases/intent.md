# intent: ux-phases

- 解きたい課題: フレームワークの上流は PRFAQ と UX調査(仮説検証)までしかなく、
  「誰の・どんな行動を・なぜ支援するか」(ユーザーストーリー)と「それをどんな体験・
  画面フローで実現するか」(UX設計)を成果物として残すフェーズが無い。UX調査の結果が
  要件(spec)に飛んでしまい、体験の設計判断が暗黙になる。
- この bolt でやること:
  1. lifecycle にユーザーストーリー/UX設計の2フェーズを追加(UX調査→ストーリー→UX設計→要件)
  2. テンプレート追加: user-story.md / ux-design.md
  3. スキル追加: user-story / ux-design(既存ミニループ・レビューゲートが掛かる形)
  4. 参照の更新: AGENTS.md 参照表・bolt スキルのフェーズ表・README マップ
- やらないこと: デザインツール連携(Figma等)、ビジュアルデザイン(見た目)のガイド、
  prfaq/ux-research スキル本体の再設計
- 完了の定義: 新2フェーズが lifecycle のフェーズ判定・bolt スキル・AGENTS.md から
  一貫して参照され、リンク検証グリーン、相互レビュー収束
